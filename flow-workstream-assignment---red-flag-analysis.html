Red Flag Key

8 analysis categories
Expected Important

Flow references fields containing the word “Expected” (e.g., “ACV (Expected)”, “ACV_Expected__c”). These may indicate placeholder fields or fields with unclear business meaning.

Trig Ctrl Critical

Flow contains Apex actions that programmatically disable/re-enable managed package triggers at runtime (e.g., CPQTriggerControlInvocable). This is a risky pattern that can cause data inconsistency.

Fin DML Critical

Flow creates, updates, or deletes financial records (invoices, payments, credit notes, billing configs). Incorrect modifications can impact revenue recognition and billing accuracy.

Formulas Important

Flow contains complex formulas (>300 chars, deep nesting, or many functions). These are hard to maintain and debug, increasing risk of calculation errors.

Hard IDs Important

Flow contains hardcoded Salesforce record IDs. These break when migrating between environments (sandbox → production) and are a deployment anti-pattern.

SOQL Loop Critical

Flow performs SOQL queries or DML operations inside a loop. This risks hitting Salesforce governor limits and causing runtime failures at scale.

Dup Query Warning

Flow queries the same object multiple times when results could potentially be reused. Indicates opportunity for optimization and reducing query consumption.

Names Warning

Flow nodes have generic or auto-generated names (e.g., “Decision”, “Loop_1”, “myDecision”). Poor naming reduces readability and makes maintenance difficult.
